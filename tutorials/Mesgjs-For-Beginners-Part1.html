Welcome to Mesgjs (pronounced "messages" — or "message J S" if that's clearer)! This tutorial is designed for people who are brand new to programming, or who are new to Mesgjs specifically. We'll go step by step, with plenty of examples you can try yourself.

Don't worry if things feel unfamiliar at first — that's completely normal. By the end of Part 1, you'll be writing real Mesgjs programs!

1. Opening the REPL and Browser Console

The easiest way to try Mesgjs is the online REPL. A REPL is simply a page where you can type code and run it immediately — no installation needed.

Open the REPL in a new tab:
Open the Mesgjs REPL ↗

You'll see a text area where you can type (or paste) Mesgjs code, and a Transpile button to run it.

Where does the output go?

Unlike some programming environments, Mesgjs doesn't print output directly on the page. Instead, output appears in the browser's developer console. Here's how to open it:

  • Windows / Linux: Press F12, then click the Console tab
  • Mac: Press Cmd + Option + I, then click the Console tab
  • Or right-click anywhere on the page and choose Inspect, then click Console

Keep the console open while you work through this tutorial — that's where you'll see the results of your programs.

You can paste any of the code examples in this tutorial directly into the REPL and click Transpile to run them. Each example has a 📋 button to copy it to your clipboard.

Exercise 1.1 — Open the REPL

  1. Open https://www.mesgjs.org/repl.html in a new tab.
  2. Open the browser developer console (see instructions above).
  3. The REPL already has a sample program in it. Click Transpile and look for output in the console.

You should see Hello, world! appear in the console. If you do — great, everything is working!

2. Two Core Concepts: Objects and Messages

Almost everything in Mesgjs comes down to just two ideas:

ConceptWhat it isReal-world analogy
Object Something that holds information and knows how to do certain things An accountant who keeps your financial records
Message A request you send to an object, asking it to do something and give you back a result Calling that accountant and asking them to prepare your tax return

In Mesgjs, every action is a message. Want to add two numbers? Send a message. Want to combine two pieces of text? Send a message. Want to display something on screen? Send a message.

The basic pattern for sending a message looks like this:

receiver(operation  optional-extra-values)

Where:

  • receiver — the object you're sending the message to
  • operation — the name of the action you want (always the first thing inside the parentheses)
  • optional-extra-values — any additional information the operation needs

That's really the heart of Mesgjs. Let's see it in action!

3. Your First Program

Here's a simple Mesgjs program. Let's look at it piece by piece:

@c(log "The answer is:" 42)

Breaking it down:

PartWhat it is
@cThe Mesgjs core object — a built-in toolbox of useful operations. Think of it as your Swiss Army knife.
( and )These parentheses mean "send a message." They always come in matching pairs.
logThe operation (the action we're requesting) — this one asks @c to write something to the console
"The answer is:"A piece of text surrounded by double-quotes. Text like this is called a string. It can contain spaces and special characters.
42A number

What the program does: it sends the log message to @c, asking it to write both "The answer is:" and 42 to the browser console.

The @ symbol at the start of @c is a special marker used by the Mesgjs language itself. You'll see a few other @-prefixed names as you learn more, like @t (meaning "true"), @f (meaning "false"), and @u (meaning "undefined" — nothing). You won't use @ at the start of your own names.

Three ways to write text

Mesgjs has three ways to write text values:

  • "double-quoted" — text between double-quotes; can contain spaces; use \" to include a literal double-quote inside
  • 'single-quoted' — works the same way; use \' to include a literal single-quote inside
  • word — a simple unquoted word with no spaces (like log, hello, firstName)

Exercise 3.1 — Hello, World!

Paste this into the REPL and click Transpile. Check the console for output.

@c(log "Hello, world!")

You should see Hello, world! in the console.

Exercise 3.2 — Multiple Values

The log operation can display several values at once, separated by spaces. Try:

@c(log My name is Alice and I am 30 years old.)

What do you see in the console? Try changing the name and age to your own.

Exercise 3.3 — Words vs. Quoted Strings

Simple words (without spaces) and quoted strings are both text values. Try logging a word and a quoted string:

@c(log hello)
@c(log "hello")

Both display hello — they're the same value, just written differently.

The important rule: if your text needs to contain a space, you must use quotes. A bare word like hello world would be read as two separate values (hello and world), not one. Try:

@c(log "hello world")

This logs a single value: hello world (though the output will look the same in this case whether logged as one value or two).

4. Mathematical Operations in Mesgjs

In Mesgjs, you don't write calculations in traditional mathematical notation like 1 + 2 or 3 × 4. Instead, you send messages to numbers asking them to do math. This might feel unusual at first, but it's very consistent — everything is a message!

Here are some of the basic math messages you can send to a number:

MessageMeaningExampleResult
(add n)Add3(add 4)7
(sub n)Subtract10(sub 3)7
(mul n)Multiply3(mul 4)12
(div n)Divide12(div 4)3

You can also pass multiple values to a math message, and it will apply them all in sequence. For example, 10(add 20 30) adds both 20 and 30 to 10, giving 60:

@c(log 10(add 20 30))

You can also use a message result as part of another message. For example, the join message attaches more text to the end of a string:

@c(log "The answer"(join " is:") 6(mul 7))

Here, "The answer"(join " is:") produces "The answer is:", and 6(mul 7) produces 42. Both results are then passed to @c(log ...). In the next section, we'll look more closely at the different ways you can combine messages like this.

Exercise 4.1 — Basic Arithmetic

Try each of these in the REPL:

@c(log 5(add 3))
@c(log 10(sub 4))
@c(log 6(mul 7))
@c(log 15(div 3))

What results do you see?

Exercise 4.2 — Multiple Values at Once

The add message can accept multiple values to add all at once. Try:

@c(log 10(add 20 30))

Now try using mul with multiple values. What does 2(mul 3 4) give you?

@c(log 2(mul 3 4))

Result: 24. The mul message multiplies 2 by 3 (giving 6), then multiplies that by 4 (giving 24).

Exercise 4.3 — Joining Text

The join message on text values attaches more text to the end. Try:

@c(log Hello(join ", " world "!"))

What does this produce? Try building your own greeting by changing the values.

5. Ways to Combine Messages

Did you notice in the previous section how we had a message inside another message — like 6(mul 7) sitting inside @c(log ...)? There are actually two different ways to combine messages, and understanding both will give you a lot of power.

Nesting (inner result feeds outer)

You can put a message inside another message's values. This is called nesting. The inner message runs first, and its result is used by the outer message.

Here's a simple example — calculating 2 + (3 × 5):

@c(log 2(add 3(mul 5)))

Reading from the inside out:

  1. 3(mul 5) — multiply 3 by 5, giving 15
  2. 2(add 15) — add 15 to 2, giving 17
  3. @c(log 17) — display 17 in the console

Here's another nesting example — calculating 9 - (2 × 3):

@c(log 9(sub 2(mul 3)))

Reading from the inside out: 2(mul 3) gives 6, then 9(sub 6) gives 3.

Chaining (left to right, one after another)

You can also write messages one after another. This is called chaining. Each message is sent to the result of the one before it:

@c(log 6(mul 7)(div 2))

Reading left to right, and inner to outer:

  1. 6(mul 7) — multiply 6 by 7, giving 42
  2. 42(div 2) — divide 42 by 2, giving 21
  3. @c(log 21) — display 21 in the console

This is (6 × 7) ÷ 2.

Chaining works with text too. Here we start with a word and build up a sentence step by step:

@c(log "Mesgjs"(join " is")(join " fun!"))

Step by step:

  1. "Mesgjs"(join " is") → "Mesgjs is"
  2. "Mesgjs is"(join " fun!") → "Mesgjs is fun!"
  3. That result is logged to the console

Nesting vs. Chaining — the key difference

TechniqueHow it worksExampleMeaning
Nesting Inner message runs first; its result is used by the outer message 9(sub 2(mul 3)) 9 − (2 × 3) = 3
Chaining Messages run left to right; each uses the result of the previous 9(sub 2)(mul 3) (9 − 2) × 3 = 21

You can also mix nesting and chaining in the same expression!

Exercise 5.1 — Calculate (9 − 2) × 3

Use sub and mul messages in a chain to calculate (9 - 2) × 3:

@c(log 9(sub 2)(mul 3))

Result: 21. The chain 9(sub 2) gives 7, then 7(mul 3) gives 21.

Exercise 5.2 — Calculate 9 − (2 × 3)

Now calculate 9 - (2 × 3) using nesting so the multiplication happens first.

@c(log 9(sub 2(mul 3)))

Result: 3. The nested 2(mul 3) gives 6, then 9(sub 6) gives 3.

Exercise 5.3 — Text Chain

Build a sentence by chaining join messages. Start with hello, join a space to it, then join world. Then, as a bonus, chain one more message to convert the whole result to uppercase using (toUpper).

Try two different approaches — one that joins the space and world in separate steps, and one that joins them both in a single join message.

Approach 1 — separate joins, then uppercase:

@c(log hello(join " ")(join world)(toUpper))

Approach 2 — join space and world together, then uppercase:

@c(log hello(join " " world)(toUpper))

Both give the same result: HELLO WORLD

In Approach 2, hello(join " " world) joins the space and world in a single message, giving "hello world", then (toUpper) converts it to uppercase.

Exercise 5.4 — Average of Three Numbers

Calculate the average of 10, 20, and 30. (Hint: add them all up, then divide by 3.)

Remember: add can accept multiple values at once — for example, 10(add 20 30) adds 20 and 30 to 10 in one step. Try both approaches!

Approach 1 — chain of adds:

@c(log 10(add 20)(add 30)(div 3))

Approach 2 — add multiple values at once:

@c(log 10(add 20 30)(div 3))

Both give the same result: 20. The second approach is more concise — 10(add 20 30) adds all three numbers in one message, giving 60, then 60(div 3) gives 20.

6. Named Values in Messages

Some messages accept named values — extra information identified by a name rather than just by position. Named values use the = sign to pair a name with a value:

@c(log first(join second third with=", "))

What's happening:

  • first, second, third are simple word values (text without spaces)
  • with=", " is a named value — the name with is paired with the value ", "
  • The join message uses the with value as a separator between all the pieces being joined
  • The result is the text "first, second, third"
When = appears between a name and a value (like with=", "), it creates a key/value association — it is not an assignment. Actual assignment happens through messages like (nset), as you'll see in the next section. Note that = can also appear as a message operation (e.g., 5(= 5) tests for equality), but that's a different use — as a message operation it's always the first thing inside the parentheses.

Exercise 6.1 — Join with a Separator

Try joining words with different separators:

@c(log one(join two three with=" - "))
@c(log one(join two three with=" | "))

What do you get for each line?

Exercise 6.2 — Build a Sentence

Use join with a space separator to build a sentence from individual words:

@c(log The(join quick brown fox with=" "))

Try building your own sentence this way.

Exercise 6.3 — More Values, Same Separator

You can join as many values as you like. Try adding more:

@c(log a(join b c d e with="-"))

What do you get?

You get: a-b-c-d-e

The join message joins the starting value (a) with all the extra values (b, c, d, e), placing the with separator between each pair.

7. Saving Values for Later: Scratch Storage

So far, every value we've used has been written directly in the code. But what if you want to calculate something once and use the result in several places? Or what if you want to give a value a meaningful name?

Mesgjs provides several storage objects for saving and retrieving values. They have very short names because they're used so often.

The # object is used for scratch storage — a temporary notepad for values you need during the current operation. Think of it like a whiteboard you can write on and erase.

Saving a value

Use #(nset name=value) to save a value with a name. The nset message (short for "named set") can save several values at once:

#(nset firstName=Alice lastName=Smith age=30)

You can also save a single value using #(set name to=value):

#(set greeting to="Hello, world!")

Retrieving a value

Use #(at name) to get a saved value back:

@c(log #(at firstName))

Because retrieving values is so common, there's a handy shortcut: just write #name and Mesgjs automatically retrieves it for you:

@c(log #firstName)

Both forms do exactly the same thing. The shortcut (#name) is much more common in practice.

A complete example

// Save some values
#(nset firstName=Alice lastName=Smith)

// Use them
@c(log My name is #firstName #lastName)

This displays: My name is Alice Smith

The // at the start of a line begins a comment — text that Mesgjs ignores completely. Comments are just notes for the person reading the code. You'll learn more about comments in Part 2.

A quick look at all the storage types

Mesgjs has several storage objects, each for a different purpose. You'll learn about all of them as you progress through the tutorials:

StorageNamePurpose
#ScratchTemporary values for the current operation (this tutorial)
%Object propertiesPersistent values belonging to an object (Part 3)
!Message parametersValues passed along with a message (Part 3)
%*Global shared storageValues shared across the entire program (Part 4)
%/Module storageValues private to a module (Part 4)

Exercise 7.1 — Save and Retrieve

Save your name and favorite number, then display them:

#(nset myName="Your Name" favoriteNumber=42)
@c(log "My name is" #myName "and my favorite number is" #favoriteNumber)

Change the values to your own name and number, then run it.

Exercise 7.2 — Save a Calculation Result

You can save the result of a calculation and use it later:

#(nset result=6(mul 7))
@c(log "6 times 7 is" #result)

Now try saving the result of (9 - 2) × 3 and displaying it.

#(nset result=9(sub 2)(mul 3))
@c(log "(9 - 2) x 3 =" #result)

Result: (9 - 2) x 3 = 21

Exercise 7.3 — Multiple Scratch Values

Save a city and country, then use them in a sentence:

#(nset city=Chicago country="United States")
@c(log #city "is in the" #country)

Try changing the city and country to somewhere you know.

Exercise 7.4 — Reusing a Value

One big benefit of saving values is that you can reuse them without retyping or recalculating. Try:

#(nset base=12)
@c(log #base "squared is" #base(mul #base))
@c(log #base "cubed is" #base(mul #base)(mul #base))

What do you get? Try changing the base value to something else.

With base=12:

  • 12 squared is 144
  • 12 cubed is 1728

Notice how #base is used as both the starting value and as a parameter: #base(mul #base) means "multiply the base by itself."

Exercise 7.5 — Shortcut vs. Full Form

Both #name and #(at name) retrieve a value. Verify they're equivalent:

#(nset x=99)
@c(log #x)
@c(log #(at x))

Both lines should display 99.

Exercise 7.6 — Building a Full Name

Save a first name and last name, then use join to combine them with a space in between:

#(nset first=Jane last=Doe)
@c(log #first(join " " #last))

Try it with your own name. Can you also display the name in "Last, First" format?

#(nset first=Jane last=Doe)
@c(log #last(join ", " #first))

Result: Doe, Jane

8. Summary

Excellent work! Here's a recap of everything you've learned in Part 1:

ConceptExampleWhat it does
Send a message@c(log "hello")Asks @c to display "hello" in the console
Number arithmetic6(mul 7)Multiplies 6 by 7, giving 42
Multiple values in one message10(add 20 30)Adds 20 and 30 to 10, giving 60
Join text"hello"(join " world")Gives "hello world"
Nesting9(sub 2(mul 3))9 − (2×3) = 3; inner runs first
Chaining6(mul 7)(div 2)(6×7) ÷ 2 = 21; left to right
Named valuesa(join b with="-")Gives "a-b"
Save to scratch#(nset x=42)Saves 42 with the name x
Retrieve from scratch#x or #(at x)Gets the saved value of x

Key things to remember

  • Message syntax is always receiver(operation values)
  • The = sign is for pairing a name with a value — it is not assignment or comparison
  • Nesting: inner message runs first; its result is used by the outer message
  • Chaining: messages run left to right; each uses the result of the previous
  • #name is a shortcut for #(at name)
  • Output goes to the browser console — keep it open!

Final Exercise — Putting It All Together

Write a short Mesgjs program that:

  1. Saves your first name, last name, and birth year in scratch storage
  2. Calculates your approximate age (use 2026 as the current year)
  3. Displays a sentence like: Hello! I'm Jane Doe and I'm about 30 years old.
#(nset firstName=Jane lastName=Doe birthYear=1996)
#(nset age=2026(sub #birthYear))
@c(log "Hello! I'm" #firstName #lastName "and I'm about" #age "years old.")
